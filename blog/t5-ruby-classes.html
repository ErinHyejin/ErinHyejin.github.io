Ruby Class Basic

Jan 22, 2015

Ruby beautifully supports object oriented programming. It allows us rebuild the real world in the codes with classes and objects. Let's look at the example code to learn new terms related to class.

Creating a class

- First, class name should be capitalized. We have 'Dog' class here.
- Inside a class, the first method is called initialize. Often times, initialize can take arguments and change them as instance variables.
- Instance variable is a variable that can be used inside of a class regardless of methods. To make an instance variable, add '@' in front of a variable. In the above image, we assigned argument name into a instance variable @name. Then, we can call @name in any method within the 'Dog' class when we want to use it in the class.

Looking at the Figure 2. How many methods are inside the 'Dog' class? There are three methods for now, but it is possible to add more methods as many as we want.
- The first method is 'initialize',
- The second one is 'bark' which prints a string of a sentence,
- The last one is 'eat' which takes an argument, item, and print a string of a sentence depending on the item name.
- These methods are called instance methods.

Creating an object and using "new"

At this point, let's see how this class works. I created an object 'mydog' and its name "Lulu." - To make a new object, at the end of the code, type
"object_name = class_name.new(arguments)"

- In this Dog class, the argument is a name of an object and we can create an object like this:
mydog = Dog.new("Lulu")

- With an object 'mydog', we can call methods in the Dog class: 'mydog.method.' In the Figure 3 line 23, I called
mydog.bark
Then the output will be
"Lulu barks: Bow wow! Woof woof!"
according to the Figure 2 line 9.

- In Figure 3 line 24, I called eat method with an item "soup." If a method inside the class need an argument (or several arguments), make sure you put an argument(arguments) when calling the method. Now you may guess the output of this code:
"soup is yummy, but I want chicken jerky."
According to the Figure 2 line 16, since the item name is not same as "chicken jerky," it returns the else statement.

Using class, it is available to create an object and use several method with ease. It will be also more useful when you build more complex codes.
